Collaborative Learning Discussion 1

Secure Software Development

17th June, 2023 John Bodden

Below, I will explain the UML modelling tool and the diagrams I would use to analyse a website's (XSS) Cross-site scripting attack and the steps carried out. I will explain what UML tools I would use and why they are my most appropriate choice.

Figure 1. Illustrates one of OWASP’s top ten vulnerabilities in a vulnerable website that an attacker has discovered and injected malicious code into. When a visitor accesses the website, the malicious script is activated each time, sending the session cookie to the malicious actor, who then gains access to the visitor’s session. (Software Testing Help, 2023)

Figure 1: One of OWASP's Top Ten Vulnerabilities unsing UML diagram. (Software Testing Help, 2023)

Visual Paradigm

“Visual Paradigm is a software design tool designed for agile software projects.” (Visual Paradigm, n.d.) has a built-in agile development methodology in the software, which helps build systems in an agile way. Users of Visual Paradigm can create many types of Unified Modelling Language (UML) diagrams with minimal effort, making it flexible for the needs of small and large teams, allowing for customisation based on individual requirements. Its ability to collaborate among team members offers various diagram types beyond the traditional UML diagrams. It integrates with other tools like GitHub, making it my choice to model the proposed software.

Sequence Diagram

The sequence diagram displays a time sequence depicting the processes and objects in a message exchange between other objects and processes during the XSS attack. In Figure 1, the sequence diagram would be one UML diagram I would use for displaying the elements or parts of a web-based application and software processes as they interact over time and are organised according to “objects (horizontally) and time (vertically)” (Visual Paradigm, n.d.) which can help to identify potential vulnerabilities in the application.

Component Diagram

Component diagrams can be used to model the components of the web application and their interactions during an XSS attack. Component diagrams can help identify the vulnerable parts of the application that the attacker can exploit to initiate an XSS attack. It can also be used to debug issues and improve communication among team members and stakeholders when the data flow is unclear, allowing the software developer to analyse the code improvement processes deeply. (Geeks for Geeks, 2022)

Class Diagram

The class diagram is another UML diagram I would use to represent the structure of the system classes and objects in a system and their relationships with each other when modelling software systems using object-oriented principles. In this phase, I can identify courses with too many responsibilities or poorly defined interfaces to ensure that software systems are well-architected, maintainable, and extensible and can be used throughout the SDLC. (IBM, 2021) Overall, a combination of these UML diagrams can effectively model and identify potential vulnerabilities in a web application that can lead to cross-site scripting attacks. This can help to design and implement countermeasures to prevent XSS attacks in the application.
